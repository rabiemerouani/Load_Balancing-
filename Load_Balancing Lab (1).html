§ Introduction — Contexte & Problématique

L'omniprésence des architectures Cloud, la 5G et les applications à forte intensité de données imposent une pression critique sur les infrastructures. La simple mise à l'échelle matérielle (scale-up) s'avère insuffisante — la clé réside dans l'équilibrage de charge intelligent.

La problématique fondamentale : comment répartir N utilisateurs sur M serveurs disposant de capacités distinctes ? Les approches centralisées classiques (Round Robin, Least Connection) montrent de graves limites : point unique de défaillance (SPoF), rigidité aveugle, asymétrie d'information. La Théorie des Jeux offre un paradigme alternatif : concevoir le réseau comme un écosystème d'agents autonomes rationnels.

📐 Formule de Coût Tri-Critère (§2.1.2 du rapport)
Cij(S) = α · dij  +  β · (Lj(S) / capj)  +  γ · (1 / (bwij + ε))
α · dij
Latence de propagation (ms)
β · (Lj/capj)
Facteur de congestion CPU
γ · (1/bwij)
Pénalité bande passante
Uij = 1/(1+Cij)
Utilité décroissante ∈ (0,1]
α,β,γ ≥ 0 sont les coefficients de pondération (normalisés α+β+γ=1). Un réseau IoT priorisera la latence (α élevé). Un réseau de stockage Cloud priorisera la bande passante (γ élevé). ε = 0.001 est la constante de régularisation anti-division par zéro (COST_BW_EPS).
🎮 Jeu de Congestion — Γ = (N, M, Ci) (§2.1.1)
Lj(S) = Σi∈N : si=j wi
Formalisé par Rosenthal (1973) : chaque joueur i ∈ {1…N} possède un poids wi ∈ ℝ⁺ (volume de trafic) et choisit un serveur j ∈ {1…M}. La charge Lj(S) est la somme des poids des joueurs affectés au serveur j. Propriété fondamentale : ces jeux possèdent toujours une fonction de potentiel qui garantit la convergence vers un équilibre pur.
⚔ Équilibre de Nash Pur (§2.1.3)
∀i ∈ N, ∀s'i ∈ M : Ci,s*(s*i, S*-i) ≤ Ci,s'(s'i, S*-i)
Un profil S* est un Équilibre de Nash Pur si aucun joueur ne peut réduire son coût par une déviation unilatérale. S*-i = vecteur des choix de tous les autres joueurs. Implémentation : Best-Response Dynamics itérative — le joueur avec le plus grand gain potentiel migre, jusqu'au point fixe. Anti-ping-pong par empreinte d'état (State Hashing) + seuil ε adaptatif.
🌐 Optimum Social (§2.1.4)
Ssocial = argminS Σi=1N Ci(S)
L'Optimum Social minimise la somme globale des coûts — configuration idéale d'un orchestrateur omniscient. Un joueur peut être sacrifié (coût individuel augmenté) si cela désengorge massivement l'infrastructure. Différence clé vs Pareto : le Social accepte les dégradations individuelles si le bien collectif s'améliore. Recherche locale stricte : chaque déplacement testé, accepté seulement si ΔC_social < 0.
✨ Optimum de Pareto (§2.1.4)
∄ S' : ∀i Ci(S') ≤ Ci(S) et ∃i strict
Un profil S est Pareto-optimal si aucune amélioration d'un joueur est possible sans dégrader un autre. Solveur Pareto : un déplacement est validé si (1) il améliore strictement le joueur qui bouge, (2) ne détériore aucun autre (ε = ±0.001), (3) ne sature pas le serveur cible (cap×1.25). Converge vers la frontière d'efficacité Pareto locale.
⚡ Prix de l'Anarchie — PoA (§2.1.5)
PoA = Σi Ci(S*) / Σi Ci(Ssocial)
Introduit par Koutsoupias & Papadimitriou. Toujours PoA ≥ 1. PoA = 1.0 → anarchie parfaite, égoïsme = optimum. PoA ∈ (1, 1.5) → perte modérée. PoA ≥ 1.6 → Dungeon Break alert. Roughgarden (2002) : PoA ≤ (3+√5)/2 ≈ 2.618 pour fonctions de coût affines.
🎯 Potentiel de Rosenthal (§1.1.3)
Φ(S) = Σj Σk=1Lj cj(k)
Fondement de la convergence Nash (Rosenthal 1973) : Φ décroît strictement à chaque Best-Response. Comme Φ est bornée inférieurement, la dynamique converge nécessairement vers un équilibre pur — excluant les oscillations infinies. Implémentation : Φ(S) = Σj [L·(α·d + γ/(bw+ε)) + β·L(L+1)/(2·cap)]. Affiché en live dans Graphes & Valeurs.
📡 QRE — Quantal Response Eq. (§2.2.2)
Pij = exp(-Cij/τ) / Σk exp(-Cik/τ)
Modélise les agents à rationalité limitée (Milchtaich 1996). La température τ contrôle le niveau de rationalité : τ → 0 : comportement déterministe, Best-Response pure. τ → ∞ : choix uniforme aléatoire. Différence vs Nash Mixte : le QRE applique la softmax sur les coûts négatifs (pas sur les utilités U=1/(1+C)). Stabilisation numérique par soustraction du min avant exp.
⚠ Paradoxe de Braess (Roughgarden·Tardos)
∃ S' ⊂ S : Nash(S') > Nash(S)
L'ajout d'un serveur ultra-attractif (latence minimale) peut dégrader le coût collectif : l'égoïsme Nash concentre tous les agents dessus, créant la saturation. L'Optimum Social l'ignore. Demonstré dans le scénario Paradoxe de Braess — 4 serveurs dont 1 "Attracteur" (lat=1ms, cap=50). Nash Pur → saturation totale. Social Local → ignore le piège.
⚖ Indice d'Équité de Jain (§2.2.4)
J(L) = (Σj Lj)² / (M · Σj Lj²)
Strictement borné J ∈ [1/M, 1]. J = 1 → équité parfaite, tous les serveurs supportent la même charge. J = 1/M → catastrophique, une seule machine supporte tout le trafic. Invariant par mise à l'échelle — permet de comparer indépendamment de la charge absolue.
📊 Makespan & Taux d'Utilisation (§2.2.4)
Makespan = maxj∈M Lj    U = Σ Lj / Σ capj × 100%
Makespan (éq. 2.9) = charge brute du serveur le plus lourdement sollicité. Minimiser le Makespan = réduire la réactivité maximale de la plateforme. Taux d'utilisation : 70–80% = dimensionnement idéal. >95% = sous-dimensionnement critique. <40% = gaspillage économique. Makespan Ratio = max(Lj/capj) normalisé.
🔄 Approches Classiques d'Équilibrage (§1.1.1)
Round Robin · Weighted RR · IP Hash · Least Connection
Round Robin : rotation séquentielle, aveugle à la charge réelle. Weighted RR : proportionnel à capj — nœuds puissants reçoivent plus de trafic. IP Hash : hash(clientId) % M → même client toujours vers même serveur (persistance de session). Least Connection : route vers le serveur avec le moins de sessions actives — glouton, oscillations possibles. Limite commune : ignorent la nature hétérogène des flux.
🌊 Selfish Routing & Roughgarden (§1.1.3)
PoA ≤ (3+√5)/2 ≈ 2.618 (fonctions affines)
Roughgarden & Tardos (2002) ont démontré que l'absence de coordination pousse les individus à saturer les axes principaux — illustrant le paradoxe de Braess. Milchtaich (1996) a étendu la théorie aux jeux de congestion hétérogènes où chaque agent possède sa propre perception du coût — fondement logique du simulateur Shadow Core. Le PoA pour fonctions affines est borné par 4/3 ≤ PoA ≤ 2.618.
